Über JSON-zu-YAML-Konverter online

Der JSON-↔-YAML-Konverter wandelt Daten in beide Richtungen zwischen JSON und YAML um, mit Validierung, vollständig in Ihrem Browser. JSON ist kompakt und in APIs allgegenwärtig; YAML ist besser lesbar und in Konfigurationsdateien (CI-Pipelines, Kubernetes, Docker Compose) verbreitet. Mit diesem Tool können Sie eine Struktur zwischen beiden verschieben, ohne von Hand zu bearbeiten, wobei Verschachtelung, Arrays und Typen erhalten bleiben.

Verwendung

  1. Fügen Sie JSON ein, um es in YAML umzuwandeln, oder fügen Sie YAML ein, um es in JSON umzuwandeln.
  2. Die umgewandelte Ausgabe erscheint sofort mit angewandter Formatierung.
  3. Ungültige Eingaben werden markiert, sodass Sie die Syntax vor der Umwandlung korrigieren können.
  4. Kopieren Sie das Ergebnis in Ihre Konfigurationsdatei oder API-Anfrage.

Häufige Fragen

Ist YAML eine Obermenge von JSON?

Ja. Jedes gültige JSON ist auch gültiges YAML, sodass in einen YAML-Parser eingefügtes JSON funktioniert. Die Umkehrung gilt nicht – YAML hat Funktionen (Kommentare, Anker, mehrere Dokumente), die JSON nicht ausdrücken kann.

Was passiert mit YAML-Kommentaren bei der Umwandlung in JSON?

Sie werden verworfen. JSON hat keine Kommentarsyntax, sodass Kommentare nicht dargestellt werden können und bei der Umwandlung in JSON verloren gehen.

Warum wurde meine YAML-Zahl zu einer Zeichenkette (oder umgekehrt)?

YAML leitet Typen aus der Formatierung ab: Zahlen ohne Anführungszeichen werden zu Zahlen und true/false zu booleschen Werten. Setzen Sie einen Wert in Anführungszeichen, um ihn als Zeichenkette zu erzwingen. Beachten Sie, dass einige YAML-1.1-Tools auch yes/no/on/off als boolesche Werte behandeln – eine häufige Überraschung beim Verschieben von Konfigurationen zwischen Tools.

Verlassen die Daten meinen Browser?

Nein. Umwandlung und Validierung laufen lokal ab – nützlich für Konfigurationsdateien, die Geheimnisse enthalten können.
