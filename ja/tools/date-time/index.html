概要： 日付・時刻フォーマット変換ツール

日付は必要な形式で届くことはめったにありません。このコンバーターは、単一の時点を Unix タイムスタンプ（秒またはミリ秒）、ISO 8601 文字列、人間が読める形式の間で変換するため、ログを読んだり、API をデバッグしたり、タイムゾーンの違いを一目で照合したりできます。すべてが完全にブラウザ内で計算されるため、日付やタイムスタンプがデバイスを離れることはありません。

使い方

  1. Unix タイムスタンプや ISO 8601 の日付文字列などの値を入力欄に貼り付けるか入力します。
  2. 入力がどの形式かを選択します（ツールは一般的な形式を自動検出することもできます）。
  3. 目的の形式を選び、関連する場合はタイムゾーン（UTC またはローカルゾーン）を選択します。
  4. 変換された結果を読み取り、コード、テスト、API リクエストで使うためにコピーします。

よくある質問

Unix タイムスタンプとは何ですか？

Unix タイムスタンプは、Unix エポック（1970-01-01T00:00:00Z、つまり1970年1月1日の UTC 午前0時）から経過した秒数です。多くのシステムはミリ秒も使い、同じ瞬間に対して1000倍大きくなります。

秒とミリ秒の違いは何ですか？

秒単位の Unix タイムスタンプは現在の日付では通常10桁ですが、ミリ秒タイムスタンプは13桁です。JavaScript の Date.now() はミリ秒を返す一方、多くのバックエンド言語は秒を使うため、変換前に桁数を確認してください。

タイムゾーンはどのように扱われますか？

Unix タイムスタンプと Z で終わる ISO 8601 文字列は絶対的で UTC に紐付いています。日付を人間が読めるローカル形式で表示するとき、コンバーターはオフセットを適用するため、同じ瞬間が異なるタイムゾーンで異なる時刻として表示されることがあります。

私のデータはサーバーに送信されますか？

いいえ。すべての解析と変換は組み込みの日付関数を使ってブラウザ内でローカルに行われるため、入力した日付やタイムスタンプがどこかにアップロードされることは一切ありません。
