À propos de Convertisseur JSON vers YAML en ligne

Le convertisseur JSON ↔ YAML transforme les données entre JSON et YAML dans les deux sens, avec validation, entièrement dans votre navigateur. Le JSON est compact et omniprésent dans les API ; le YAML est plus lisible et courant dans les fichiers de configuration (pipelines CI, Kubernetes, Docker Compose). Cet outil vous permet de déplacer une structure entre les deux sans édition manuelle, en préservant l'imbrication, les tableaux et les types.

Comment utiliser

  1. Collez du JSON pour le convertir en YAML, ou collez du YAML pour le convertir en JSON.
  2. La sortie convertie apparaît instantanément avec la mise en forme appliquée.
  3. Une entrée invalide est signalée pour que vous puissiez corriger la syntaxe avant la conversion.
  4. Copiez le résultat dans votre fichier de configuration ou votre requête d'API.

Questions fréquentes

Le YAML est-il un sur-ensemble du JSON ?

Oui. Tout JSON valide est aussi du YAML valide, donc du JSON collé dans un analyseur YAML fonctionne. L'inverse n'est pas vrai — le YAML possède des fonctionnalités (commentaires, ancres, documents multiples) que le JSON ne peut pas exprimer.

Qu'advient-il des commentaires YAML lors de la conversion en JSON ?

Ils sont supprimés. Le JSON n'a pas de syntaxe de commentaire, donc les commentaires ne peuvent pas être représentés et sont perdus lors de la conversion en JSON.

Pourquoi mon nombre YAML est-il devenu une chaîne (ou inversement) ?

Le YAML déduit les types de la mise en forme : les nombres sans guillemets deviennent des nombres et true/false deviennent des booléens. Mettez une valeur entre guillemets pour la forcer à rester une chaîne. Notez que certains outils YAML 1.1 traitent aussi yes/no/on/off comme des booléens — une surprise courante lors du déplacement de configurations entre outils.

Les données quittent-elles mon navigateur ?

Non. La conversion et la validation s'exécutent localement — utile pour les fichiers de configuration qui peuvent contenir des secrets.
