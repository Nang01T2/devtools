描述

在多种格式之间转换日期和时间戳，包括Unix时间戳、ISO字符串和自定义模式。适用于调试和API集成。

关于 日期与时间格式转换器

日期很少以您需要的格式出现。本转换器在 Unix 时间戳（以秒或毫秒为单位）、ISO 8601 字符串和人类可读格式之间转换某一个时刻，让您一目了然地读取日志、调试 API 并协调时区差异。一切都完全在您的浏览器中计算，因此没有任何日期或时间戳会离开您的设备。

使用方法

  1. 将一个值粘贴或键入到输入框中，例如 Unix 时间戳或 ISO 8601 日期字符串。
  2. 选择您输入所属的格式（本工具也可以自动检测常见格式）。
  3. 选择目标格式，以及相关时区（UTC 或您的本地时区）。
  4. 读取转换后的结果并复制它，以用于您的代码、测试或 API 请求。

常见问题

什么是 Unix 时间戳？

Unix 时间戳是自 Unix 纪元（即 1970-01-01T00:00:00Z，1970 年 1 月 1 日午夜 UTC）以来经过的秒数。许多系统也使用毫秒，对于同一个时刻，毫秒值是秒值的 1000 倍。

秒和毫秒有什么区别？

以秒为单位的 Unix 时间戳对于当前日期通常是 10 位数，而毫秒时间戳是 13 位数。JavaScript 的 Date.now() 返回毫秒，而许多后端语言使用秒，因此转换前请检查数量级。

时区是如何处理的？

Unix 时间戳和以 Z 结尾的 ISO 8601 字符串是绝对的，并与 UTC 关联。当您以人类可读的本地格式显示日期时，转换器会应用一个偏移量，因此同一个时刻在不同时区可以显示不同的钟表时间。

我的数据会被发送到服务器吗？

不会。所有解析和转换都使用内置日期函数在您的浏览器中本地进行，因此您输入的日期和时间戳绝不会被上传到任何地方。
