Über Base64-Encoder und -Decoder online

Der Base64-Encoder/-Decoder konvertiert Text und Binärdaten direkt in Ihrem Browser von und nach Base64. Base64 stellt Binärdaten als ASCII-Text dar, wodurch sie sicher in URLs, JSON, XML, E-Mail-Anhängen und Data-URIs eingebettet werden können. Codieren oder decodieren Sie sofort mit integrierter Validierung – nichts wird an einen Server gesendet.

Verwendung

  1. Wählen Sie, ob Ihre Quelle reiner Text oder eine Datei ist.
  2. Zum Codieren tippen oder fügen Sie Text ein (oder laden eine Datei hoch) – die Base64-Ausgabe erscheint sofort.
  3. Zum Decodieren fügen Sie eine Base64-Zeichenkette ein; der decodierte Text wird angezeigt und validiert.
  4. Kopieren Sie das Ergebnis oder laden Sie es bei der Arbeit mit Dateien herunter.

Häufige Fragen

Ist Base64 eine Verschlüsselung?

Nein. Base64 ist eine Codierung, keine Verschlüsselung. Jeder kann es decodieren – es bietet keine Sicherheit oder Vertraulichkeit. Verwenden Sie es für den sicheren Transport von Binärdaten, nicht zum Schutz von Geheimnissen.

Warum ist die Base64-Ausgabe länger als die Eingabe?

Base64 stellt jeweils 3 Bytes der Eingabe als 4 ASCII-Zeichen dar, sodass die Ausgabe etwa 33 % größer ist als die ursprünglichen Daten. Dieser Mehraufwand ist der Kompromiss für den textsicheren Transport.

Was ist ein Data-URI?

Ein Data-URI bettet eine Datei direkt in eine URL mittels Base64 ein, z. B. data:image/png;base64,iVBORw0K… Damit können Sie kleine Bilder oder Schriften ohne separate HTTP-Anfrage einbinden.

Findet das Decodieren auf einem Server statt?

Nein. Das Codieren und Decodieren läuft vollständig in Ihrem Browser ab, sodass selbst sensible Daten auf Ihrem Gerät bleiben.
