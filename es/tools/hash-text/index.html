Acerca de Generador de hash de texto

El Generador de hash calcula resúmenes criptográficos de texto o archivos usando MD5, SHA-1, SHA-224, SHA-256, SHA-384, SHA-512, SHA-3 y RIPEMD-160, todo en tu navegador. Un hash es una huella digital de longitud fija de tu entrada: la misma entrada siempre produce el mismo resumen, y hasta un cambio de un solo carácter produce uno completamente distinto. Úsalo para verificar la integridad de archivos, comparar descargas con checksums publicados o almacenar huellas de contraseñas.

Cómo usar

  1. Escribe o pega tu texto en la entrada (o arrastra un archivo).
  2. Lee el resumen de cada algoritmo en la lista de salida: se actualizan a medida que escribes.
  3. Activa las mayúsculas si tu checksum de referencia usa hexadecimal en mayúsculas.
  4. Copia el resumen que necesites y compáralo con el valor esperado.

Preguntas frecuentes

¿Qué algoritmo de hash debo usar?

Para comprobaciones de integridad, SHA-256 es el estándar moderno por defecto. Evita MD5 y SHA-1 para fines de seguridad, ya que están comprometidos frente a ataques de colisión, pero siguen siendo válidos para checksums sin fines de seguridad y para compatibilidad con sistemas heredados.

¿Puedo revertir un hash al texto original?

No. El hash es unidireccional por diseño. La única forma de 'revertir' un hash es adivinar entradas y aplicarles el hash hasta que una coincida, razón por la cual se usan hashes lentos y con sal para las contraseñas.

¿Se envía mi entrada a algún lugar?

No. Todo el cálculo de hash se ejecuta localmente en tu navegador, así que el texto y los archivos nunca salen de tu dispositivo.

¿Por qué dos herramientas dan hashes distintos para el mismo texto?

Por lo general, por un salto de línea final, una codificación de caracteres distinta o diferencias en los finales de línea (CRLF frente a LF). Los hashes son exactos a nivel de byte, así que incluso diferencias invisibles cambian el resultado.
